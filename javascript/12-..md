# 12장. 함수

## 함수를 사용하는 이유

중복된 코드들을 따로 함수에 정의를 해놓고 함수호출로 여러번 사용할 수 있다. 따라서 함수를 재사용할 수 있다. 함수를 사용하지 않는다면 중복된 코드가 많아지며 가독성이 나빠지고 실수할 확률이 높아진다. 중복된 코드들 마다 수정을 해야하기때문에 유지보수에도 좋지않다.&#x20;

<mark style="color:red;">**코드의 중복을 억제하고 재사용을 높이는 함수는 유지보수 편의성을 높이고 실수를 줄여 코드의 신뢰성을 높인다.**</mark>



## 함수

자바스크립트의 <mark style="color:red;">**함수는 객체다.**</mark> 일반적인 객체와는 달리 함수는 호출을 할 수 있는 객체다. 함수를 호출할 때는 함수 이름이 아니라 함수 객체를 가르키는 식별자를 사용해야한다. 이때 자바스크립트 엔진이 암묵적으로 함수 이름과 동일한 식별자를 암묵적으로 생성하고, 함수 객체를 할당한다.



## 함수 선언문

function 이라는 키워드와 함수 이름을 넣어서 함수를 정의할 수 있다. 함수 선언문은 <mark style="color:red;">**표현식이 아닌 문**</mark>이다. 따라서 값으로 평가될 수 없다.

```javascript
// 함수 선언문
function foo(){
    console.log('함수 선언문!')
}
```



## 함수 표현식

함수 표현식은 함수 선언문과 동일한 문법으로 사용이 가능하고, 함수 이름을 넣어도 되고 안넣어도 된다. 일반적으로 표현식에는 이름을 넣지는 않는다. 함수 표현식은 값으로 평가되는데 그 이유는 <mark style="color:red;">**자바스크립트 엔진이 코드의 문맥에 따라 함수 선언문으로 해석하는 경우와 함수 표현식으로 해석하는 경우가 있기 때문**</mark>이다.&#x20;

```javascript
// 함수 선언문
// 값으로 평가되는 문맥이 아니다. 그래서 함수 선언문으로 자바스크립트가 인식한다.
function foo(){
    console.log('함수 선언문')
}

// 밑에 함수는 const에 값을 할당하는것이기 때문에 표현식으로 해석한다.
const expression = function foo2(){
    console.log('함수 표현식')
}

// 일반적으로 함수 표현식은 위처럼 함수 이름을 사용하지않는다.
const ex = function (){
    console.log('함수 표현식은 이렇게 사용해요.')
}
```



## 함수 선언문과 함수 표현식의 차이

함수 선언문으로 정의한 함수는 함수 선언 이전에 호출할 수 있다. 함수 선언문은 함수 호이스팅이 발생하기때문에 함수 선언 이전에 호출해도 전혀 문제없지만, 함수 표현식은 값으로 평가되는 표현식이기때문에 변수에 할당을 하게되면 변수 호이스팅이 발생한다.

<mark style="color:red;">**함수 선언문은 함수 호이스팅이 발생, 함수 표현식은 변수 호이스팅이 발생한다.**</mark>

```javascript
foo() // '함수 선언문'
foo2() // 할당문을 만나기 이전에는 undefined라서 호출할 수 없기에 에러발생.

// 함수 선언문
function foo(){
    console.log('함수 선언문')
}

// 함수 표현식
var foo2 = function (){
    console.log('함수 표현식')
}
```

> 함수 호이스팅은 함수를 호출하기 전에 반드시 함수를 선언해야 한다는 당연한 규칙을 무시한다. 이같은 문제 때문에 JavaScript: The Good Parts의 저자 더글라스 크락포드는 <mark style="color:red;">**함수 선언문 대신 함수 표현식을 사용할 것을 권장**</mark>한다.

## <mark style="color:red;">화살표 함수</mark>

화살표 함수는 function 키워드 대신에 화살표 => 를 사용해서 간결하게 정의할 수 있다. 화살표 함수는 반드시 익명함수로 정의해야한다.

```javascript
// 화살표 함수

// 함수 몸체에 실행줄이 1줄일 때 {}을 생략해도 되고 return도 생략된다.
const arrow = () => console.log('화살표 함수 정의하기.')

// 매개변수가 1개일 때 소괄호를 빼도된다.
const printX = x => console.log(x);

// 매개변수가 없거나, 2개 이상일 때에는 소괄호를 써야한다.
const add = (x, y) => x + y;

// 함수 몸체에 명령줄이 2줄 이상일 때
const calculate = (x, y) => {
    const result = x + y;
    return { result, x, y }
}
```



## 함수 호출

함수를 가리키는 식별자와 소괄호로 함수를 호출 할 수 있다. 함수를 호출하면 현재의 실행 흐름을 중단하고 호출된 함수로 실행 흐름을 옮긴다.&#x20;

### 매개변수와 인수

매개변수는 함수이름 옆에 소괄호를 말하고 매개변수는 함수 몸체 안에서만 사용할 수 있는 변수다. 매개변수의 이름은 자유롭게 해도된다. 대신에 의미있는걸로 해야 가독성을 높이고 유지보수에도 용이하다.

인수는 함수를 호출할때 소괄호를 말하고 인수를 통해서 매개변수로 전달된다.

```javascript
function addNumber(x, y){ // <= x, y가 매개변수이고, 함수 몸체에서만 사용할 변수다.
    console.log(x, y, x + y) // 3, 5, 8
}

addNumber(3, 5) // <= 함수를 호출해서 사용가능하고 값으로 평가될 수 있는 표현식이여야한다.

// ========================================================================

// 인수에는 값이 하나들어있고, 매개변수에는 두개가 들어있다. 매개변수에 인수가 순서대로 할당된다.
// x는 3이고 y는 아무런 값도 받지 못했기때문에 y는 undefined로 초기화된다.
function addNumber(x, y){
    console.log(x, y) // 3, undefined
}

addNumber(3);
```



### 매개변수의 최대 개수

자바스크립트에서는 매개변수의 최대 개수가 제한되어있지 않지만, 가급적이면 유지보수성과, 클린코드(가독성)을 위해서 <mark style="color:red;">**매개변수 개수를 3개 이상 넘지않게 하는것을 권장**</mark>하고, <mark style="color:red;">**함수는 한가지의 일만 할 수 있게 만드는게 가장 좋다.**</mark>  만약에 매개변수 개수가 3개 이상 넘어간다면 하나의 객체로 만들어서 사용하는것도 좋은 방법이다.



## 반환문

반환문은 함수 몸체안에서만 사용가능하고, 함수는 return 키워드와 표현식으로 이뤄진 반환문을 사용해 실행 결과를 반환할 수 있다. 반환문을 만나면 이후에 코드는 실행되지않고 종료된다.&#x20;

```javascript
function addNumber(x, y){
    return x + y // return으로 반환했기에 함수가 종료된다.
    console.log(x + y) // 함수가 종료되어서 이 코드는 실행되지 않는다.
}

// addNumber에 return을 사용해서 x + y의 값을 반환한다. 결국 addNumber함수는 x + y의 값이된다.
const n = addNumber(5, 2); // 7

// 반환하는 값이 없다면, 함수는 undefined값을 갖게된다.
function foo(){
    return; 
}

// 함수에 return을 넣지않으면 이것 역시 undefined의 값을 갖게된다.
function empty(){ 

}

```

## 다양한 함수의 형태

### 즉시 실행 함수 (IIFE)

즉시 실행 함수는 익명함수(이름이 없는 함수)로 정의해야하고, 단 한번만 호출이 가능하다.

```javascript
(function (){
    // 코드입력
    console.log('즉시 실행 함수')
})()

// Reference Error => ()그룹 연산자에서 함수 이름을 넣게되면 함수 리터럴(표현식)으로 판단한다.
// 함수 이름은 함수 몸체 안에서만 사용 가능하기때문에 에러가 발생한다.
(function name(){
    
})

name()
```



### 재귀 함수

함수몸체 안에서 자기 자신을 호출하는것을 재귀 함수다. 재귀 함수는 반복되는 처리를 위해 사용한다. 재귀 함수는 자신을 무한으로 호출하기 때문에 무조건 탈출조건이 있어야한다. 없으면 스택 오버플로 에러가 발생한다.

```javascript
function increase(n){
  if(n < 10){
    console.log(n);
    increase(n + 1);
  }
}

increase(1);
```



### 중첩 함수(Nested Function)

함수 내부에 정의된 함수를 중첩 함수 또는 내부 함수라고 한다. 중첨 함수를 포함하는 함수는 외부 함수라 부르고 자신을 포함하는 외부 함수를 돕는 헬퍼 함수의 역할을 한다.

```javascript
// 계산을 하는 함수 안에서 더하기, 곱하기 기능이있다.
function calculate(x, y){
   function add(){
      return x + y
   }
   
   function multiple(){
      return x * y
   }
   
   return { 
      add: add(),
      multiple: multiple() 
   }
}

calulate(5, 5);
```



### <mark style="color:red;">콜백 함수(callback function)</mark>

함수의 매개변수를 통해 전달받은 함수를 콜백 함수라고 한다. 사용하는 이유는 공통된 로직은 내버려두고 새로운 기능을 추가할 때 유용하다. 또한 브라우저 API인 setTimeout 비동기에서 사용한다.

1. 매개변수를 통해 전달받은 함수를 콜백 함수라고한다.
2. 어떤 이벤트에 의해 실행되는 함수를 콜백 함수라고한다.

```javascript

// 지금도 예제가 짧아서 조건문을 넣어도되지만 인사하는 문구가 많아지면 많아질수록 조건식으로 대처하기가 조금 힘들어진다.
// switch문을 사용하더라도 함수의 코드가 계속 길어진다. 인사하는 각 함수들을 만들어주는게 더 좋다.
function greeting(first, last, callback){
  callback(first, last);
}

function goodBye(first, last){
  return `${last + first}님 안녕히가세요.`;
}

function hello(first, last) {
  return `${last + first}님 어세오세요.`;
}

greeting('DongHyun', 'Lee', hello);
greeting("DongHyun", "Lee", goodBye);

// setTimeout 비동기
// 아래코드는 1초 후에 콜백함수가 호출된다.
setTimeout(function(){ // 익명 함수로 정의한것이 콜백 함수다.
  console.log('1초 후에 실행')
}, 1000);

```

고차함수는 콜백함수를 전달받은 함수인것이다.&#x20;

위에 예제를 다시보면 콜백함수를 전달받은 greeting함수를 고차함수라고 한다.



## 순수 함수(Pure)와 비순수(Impure) 함수

### <mark style="color:red;">순수 함수</mark>

순수 함수란 외부 상태에 의존하지않고 변경하지도 않는 부수효과가 없는 함수를 말한다. 즉 순수 함수는 어떤 외부 상태에도 의존하지않고 매개변수를 통해 함수 내부로 전달된 인수에게만 의존해 값을 생성한다.

❗️순수 함수는 언제나 동일한 인풋(값)을 넣으면 동일한 아웃풋(결과)을 반환한다.

```javascript
const user = {
    name: 'dongs',
    email: 'dongs@gmail.com'
}

function pureUser(user){
    // user객체를 변경하지 않고, 깊은복사를 해서 새롭게 만들었다. 즉 외부상태에 값을 바꾸지 않았다.
    return { ...user, name: 'DongHyun' }
}

pureUser(user)
```



### 비순수 함수

비순수 함수는 외부 상태에 값을 변경하는 부수 효과가 있는 함수를 말한다.  함수 내부에서 다른 외부 상태에 값을 변경하면 이것은 비순수 함수인것이다.

```javascript
let x = 10;
let y = 20;

function setNumber(){
    x = 50;
    y = 30;
    
    return x + y;
}
console.log(x, y) // 10, 20
setNumber();
console.log(x, y) // 50, 30
```

비순수 함수를 사용하게되면 함수의 외부상태를 변경하기때문에 상태 변화를 추적하기 어렵다. 비순수 함수를 최대한 줄여서 부수 효과를 억제하는것이 좋다.&#x20;



## <mark style="color:red;">함수형 프로그래밍</mark>

함수형 프로그래밍은 순수 함수와 보조 함수의 조합을 통해 <mark style="color:red;">**외부 상태를 변경하는 부수 효과를 최소화해서 불변성을 지향**</mark>하는 프로그래밍 패러다임이다. 로직내에 존재하는 조건문과 반복문을 제거해서 복잡성을 해결하고 변수 사용을 억제하거나 생명주기를 최소화해서 상태 변경을 피해 오류를 최소화하는 것을 목표로한다.

<mark style="color:red;">**즉 순수 함수를 통해 부수 효과를 최대한 억제해 오류를 피하고 프로그램의 안정성을 높이려는 노력이다.**</mark>
